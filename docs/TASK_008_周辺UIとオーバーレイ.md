# TASK_008_周辺UIとオーバーレイ.md（プロト仕上げ）

## 目的

採用決定済みの周辺UI（ラン中メニュー／設定／ギャラリー）を実装し、プロトのUI一式を完成させる。

## 前提

- TASK_007完了：ゲーム本体は完成状態
- 対象：6画面のうち未実装分＋オーバーレイ2種

## 実装内容

1. **ラン中メニュー（オーバーレイ）**
   - バトル画面右上のボタン or ESCキーで開閉。開いてる間ゲームは一時停止
   - 内容：装備チャーム一覧（効果説明付き）／役一覧表（役名・条件・倍率の静的表）／設定を開く／リタイア（確認ダイアログ→結果画面へ）
2. **設定（オーバーレイ）**
   - BGM音量／SE音量（スライダー、localStorage等でなくメモリ保持＋ファイル保存はElectron側で単純JSONに）
   - 演出速度：通常／高速（当たり演出のtween時間を半分にする係数）
   - 画面モード：ウィンドウ／フルスクリーン切替
   - タイトル・ロビー・ラン中メニューから呼出可能
3. **ギャラリー（galleryScene.js・最小実装）**
   - CGサムネイル枠1個（未解放時はシルエット＋「???」）
   - 解放条件は仮：デバッグフラグで解放/未解放を切替できればよい（好感度システムはプロト対象外）
   - クリックで全画面表示、クリックで戻る
4. **役一覧表のデータ化**
   - src/data/hands_reference.json に役名・条件・倍率を定義し、ラン中メニューの役一覧表とダメージ計算の両方が同じデータを参照する（表示とロジックの不整合防止）

## 動作確認

- バトル中にメニューを開くとゲームが止まり、閉じると再開する
- メニュー中に敵ターンが進行しない（一時停止の漏れがないか）
- リタイアで結果画面に飛び、チップが精算される
- 演出速度「高速」で当たり演出が短くなる
- 音量スライダーが即時反映される
- ギャラリーの解放/未解放表示が切り替わる

## 注意事項

- localStorageは使わない（Electron環境での確実性のため、設定はメインプロセス経由のJSONファイル保存）
- 一時停止はPhaserのScene.pause系を使い、tweenやタイマーの取り残しに注意
- 本TASK完了で全6画面＋オーバーレイが揃い、プロトのUI一式完成
- 完了時コミット：`feat: ラン中メニュー・設定・ギャラリー`
