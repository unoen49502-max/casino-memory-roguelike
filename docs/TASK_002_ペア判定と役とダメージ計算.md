# TASK_002_ペア判定と役とダメージ計算.md（Step2前半）

## 目的

ペア判定・役判定・ダメージ計算のゲームロジックを実装し、めくったカードがペアなら消え、ダメージ数値が算出される状態にする。

## 前提

- TASK_001完了：4×4盤面、2枚めくり→1秒後に自動で裏に戻る仮ルールが動作中
- 本TASKから「ペアなら消える／不一致なら戻る」の本ルールに置き換える

## 設計原則（本TASK以降ずっと適用）

- **ロジックは src/logic/ に純粋関数で分離**。Phaser非依存（import不要）にする。理由：後のバランス調整・ダメージシミュレーター・テストで再利用するため。見た目はマスターの指示で頻繁に変わる想定なので、Scene側とロジック側を疎結合に保つ
- **ロジック→演出はイベント通知**。battleSceneはロジックの結果イベント（pair_success, pair_fail, damage_dealt等）を受けて表示するだけの構造にする

## 実装内容

1. **カード値の定義**（src/logic/card_values.js）
   - 2〜10=額面、J=11、Q=12、K=13、A=14
2. **ペア判定**（src/logic/pair_logic.js）
   - 2枚の数字が一致→ペア成立、カードは盤面から消滅（状態:消滅）
   - 不一致→1秒表示後に裏へ戻る（TASK_001の挙動を流用）
3. **役判定**（src/logic/hand_logic.js）— 1ターン内の成立ペア群から判定
   | 役 | 条件 | 倍率 |
   |---|---|---|
   | ペア | 1ペア | ×1.0 |
   | スートペア | 同数字＋同スート | ×1.5 |
   | ダブルペア | 1ターン2ペア | ×2.0 |
   | トリプルペア | 1ターン3ペア | ×3.0 |
   | ストレートペア | 連番ペアを連続成立 | ×2.5 |
   | フラッシュペア | 同スートのペアを複数成立 | ×2.5 |
   - 複数該当時は最高倍率を採用
4. **コンボ倍率**（src/logic/combo_logic.js）
   - 連続ペア成功ターンごとに +0.25（1.0→1.25→1.5…）
   - 1ペアも成立しなかったターンでリセット
5. **ダメージ計算**（src/logic/damage_calc.js）
   - `ダメージ = Σペアのカード値 × 役倍率 × コンボ倍率`（小数切り上げ）
   - 将来チャーム倍率が乗る前提で、倍率をオブジェクトで受け取る設計にしておく
6. **battleSceneへの組込**
   - ペア成立時：カード消滅＋ダメージ数値をポップアップ表示（仮演出でよい）
   - 全カード消滅時：「BOARD CLEAR」表示（勝敗はTASK_003で実装）

## 動作確認

- 同数字2枚でカードが消え、ダメージ数値が表示される
- ♠7と♠7（同スート）で1.5倍になっている
- 連続ターン成功でコンボ倍率が上がり、失敗ターンでリセットされる
- 不一致時は従来通り裏に戻る
- src/logic/ 配下のファイルがPhaserをimportしていない
