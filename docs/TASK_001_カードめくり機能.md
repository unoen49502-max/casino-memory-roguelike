# TASK_001_カードめくり機能.md（Step1）

## 目的

battleSceneに4×4の盤面を描画し、カードをクリックでめくれるようにする。フリップ演出の「手触り」を検証するのが狙い。

## 前提

- TASK_000完了済み：bootScene → titleScene → battleScene の遷移が動く
- 既存仕様：battleSceneは現在テキスト表示のみの空Scene
- 正式なカード素材はまだない。**プレースホルダーをPhaserのGraphics／Textで描画**する（裏面：暗色の角丸矩形＋簡単な模様、表面：白背景にスート記号と数字）

## 実装内容

1. **カードデータ生成ロジック**（src/objects/ もしくは src/utils/ に分離、Scene非依存の純粋関数で）
   - 13種の数字（A,2〜10,J,Q,K）から8種をランダム選出
   - 各数字2枚ずつ計16枚。スートは♠♥♦♣からランダム付与（同数字の2枚が同スートである必要はない）
   - シャッフルして4×4に対応する配列を返す
2. **Cardクラス**（src/objects/card.js）
   - 状態：裏／表／（将来用に）消滅
   - フリップアニメ：tweenで scaleX 1→0（0.15秒）→表裏テクスチャ差替→scaleX 0→1（0.15秒）、計0.3秒
   - クリック判定はカード単位。アニメ中は入力を受け付けない
3. **盤面描画**（battleScene）
   - 4×4グリッドをレイアウト。カードサイズ・余白は定数化（後の素材差し替え・盤面拡張を見据える）
   - 盤面は画面の下側2/3に配置。上側1/3は将来のキャラ立ち絵・敵HP表示用に空けておく
4. **めくりの仮ルール**（Step1限定の動作）
   - 1度に2枚まで表にできる。2枚表になったら1秒間表示し、自動で2枚とも裏に戻る
   - ペア判定・消滅・ダメージは**実装しない**（Step2の範囲）
5. カードにホバーした際、カードがわずかに浮く（y座標-4px、tween 0.1秒）

## 動作確認

- 16枚のカードが4×4で表示される
- クリックでフリップアニメが再生され、スートと数字が見える
- 2枚めくると1秒後に自動で両方裏に戻る
- フリップアニメ中の連打で表示が壊れない（3枚以上表にならない）
- 再起動するたびに数字の組み合わせと配置が変わる

## 注意事項

- titleScene・bootScene・Electron設定（main.js）は触らない
- ペア判定・役・ダメージ・敵表示は実装しない（先行実装禁止）
- カードの見た目はプレースホルダーで良いが、**テクスチャ差し替えで正式素材に置換できる構造**にしておくこと（描画処理をCardクラス内に閉じる）
- 完了時コミット：`feat: カードめくり機能＋盤面描画`
