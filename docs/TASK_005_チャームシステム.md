# TASK_005_チャームシステム.md（Step4）

## 目的

チャーム（常時発動パッシブ）の基盤と、プロト用11個のうち代表3個を実装する。

## 前提

- TASK_004完了
- チャームは「装備してる間、常時自動発動」。装備スロットは3個固定（プロトでは拡張なし）

## 実装内容

1. **チャーム定義のJSON化**（src/data/charms.json）
   - id / 名前 / レアリティ / カテゴリ / 効果タイプ / 効果値 / 説明文
   - プロト用11個を全て定義だけ先に入れる（実装は本TASKで3個、残りはTASK_007）
2. **効果適用の仕組み**（src/logic/charm_effects.js）
   - 効果タイプごとのフック方式：ダメージ計算前（倍率加算）／ターン開始時／ペア成立時／ペア失敗時 の4フックを用意
   - ダメージ計算式に組込：`ダメージ = Σペア値 × 役倍率 × コンボ倍率 × チャーム乗算補正 + チャーム加算補正`
3. **本TASKで実装する3個**（各カテゴリの代表）
   | チャーム | カテゴリ | 効果 | フック |
   |---|---|---|---|
   | ♠のブローチ | 火力 | ♠ペア成功時ダメージ+50% | ダメージ計算前 |
   | 千里眼のダイス | 記憶補助 | ターン開始時ランダム1枚を2秒表にする | ターン開始時 |
   | ストリークダイス | コンボ | 連続成功の倍率増分を+0.25→+0.5に | ダメージ計算前 |
4. **デバッグ装備画面**
   - 起動時にチャーム3個を選んで装備できる仮UI（正式なショップ入手はTASK_006）
5. **装備中チャームの表示**
   - 画面端にアイコン枠3個（仮アイコンでよい）。ホバー/タップで効果説明表示

## 動作確認

- ♠ペアのダメージが1.5倍になっている（ログで数値確認できること）
- 千里眼装備でターン開始時に1枚めくれて2秒後に戻る
- ストリーク装備でコンボ増分が倍になっている
- 3個同時装備で全部が同時に機能する
- charms.jsonの効果値を書き換えると挙動が変わる

## 注意事項

- ダメージ計算のログ出力（内訳：ペア値/役/コンボ/チャーム）をコンソールに出すこと。バランス調整で必須になる
- 効果の重複計算バグに注意（乗算は乗算同士、加算は加算同士でまとめてから合成）
- 完了時コミット：`feat: チャームシステム基盤＋3個`
