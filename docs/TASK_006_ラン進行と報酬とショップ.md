# TASK_006_ラン進行と報酬とショップ.md（Step5）

## 目的

「雑魚3戦＋ボス1戦」のラン構造、小報酬3択、ショップ、結果画面を実装し、1ランがロビーから通しで回る状態にする。

## 前提

- TASK_005完了：バトル＋チャームが動作
- プロトのラン構成：雑魚A→雑魚B→（ショップ）→雑魚C→ボスリリム
- チップ：雑魚勝利+30／ボス勝利+80（TASK_003実装済みの値を使用）

## 実装内容

1. **ラン状態管理**（src/logic/run_manager.js）
   - 進行位置／プレイヤーHP（バトル間で持ち越し）／所持チップ／装備チャームを一元管理
   - ラン開始時に初期化（HP50・チップ0・チャーム空）
2. **ロビー（最小実装）**（lobbyScene.js）
   - 「ラン開始」ボタンのみ。会話・永続要素はプロト対象外
3. **小報酬3択（雑魚戦勝利後・オーバーレイ）**
   - 3軸から1つずつ提示：
   | 軸 | 内容 |
   |---|---|
   | 即時 | HP+10回復 |
   | 経済 | チップ+40 |
   | 成長 | ラン中めくり回数+1（1ラン1回まで出現） |
   - 選択肢はsrc/data/rewards.jsonで定義
4. **ショップ（雑魚2戦目後に1回・shopScene.js）**
   - チャーム3種提示（未実装チャームは「SOLD OUT」表示で枠だけ見せる）
   - 価格：Common 40／Uncommon 70／Rare 110
   - HP回復+15＝30チップ
   - 購入するとrun_managerの装備に追加（スロット3個まで、満杯時は入替UI）
5. **結果画面**（resultScene.js）
   - 勝敗／獲得チップ合計／今回の装備チャーム一覧を表示
   - ボタン2つ：「ロビーへ」「もう1ラン直行」（run_manager初期化して即ラン再開）
6. **敗北時の扱い**
   - どのバトルで負けても結果画面へ。チップ→お菓子変換は**表示だけ**（「お菓子 +XX（未実装）」のプレースホルダー文言）

## 動作確認

- ロビー→雑魚A→報酬→雑魚B→報酬→ショップ→雑魚C→報酬→ボス→結果画面まで通しで動く
- HPがバトル間で持ち越されている
- ショップで買ったチャームが直後のバトルで機能している
- 「もう1ラン直行」でロビーを経由せず新ランが始まり、状態がリセットされている
- 途中敗北でも結果画面に正しく遷移する

## 注意事項

- Scene間のデータ受け渡しはrun_manager経由に統一（Scene間の直接参照禁止）
- 1ランの所要時間を計測してコンソールに出す（目標5〜8分の検証用）
- 完了時コミット：`feat: ラン進行・報酬・ショップ`
